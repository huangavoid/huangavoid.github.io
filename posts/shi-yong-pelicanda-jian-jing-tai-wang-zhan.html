生成HTML

搭建环境

详细说明，请参考Pelican官方网站，以及官方文档。

  1. 安装Pelican

    pip install pelican

  2. 安装Markdown支持库，以及配置文件的排版增强支持库。

    pip install markdown
pip install typogrify

  3. 安装用于自动化部署的Fabric支持库

    pip install fabric

创建应用

pelican-quickstart

生成目录如下：

yourproject/
| - content
|    | - (pages)
| - output
| - develop_server.sh
| - fabfile.py
| - Makefile
| - pelicanconf.py        # 主配置文件
| - publishconf.py       # 部署时的配置文件

使用主题

vim pelicanconfig.py
THEME = 'themes-path/theme-name'

Github上的Pelican主题

添加插件

vim pelicanconfig.py
PLUGIN_PATHS = ['plugins-path']
PLUGINS = ['plugin-name-1', 'plugin-name-2']

Github上的Pelican插件

编写内容

content
| - images
|    | - goodimages.jpg
| - downloads
|    | - goodthings.zip
| - pages
     | - test.md

vim content/pages/test.md

Title: My super title

Category: Python
Tags: pelican, publishing

![Image Text]({filename}/images/goodimages.jpg)
[Download Text]({attch}/downloads/goodthings.zip)

生成并启动

pelican -o output/path -s setting/path/pelicanconfig.py
cd output/path
python -m SimpleHTTPServer 8000

可使用Fabric或Makefile实现自动化生成、启动、部署等。

fab build
fab serve

部署

可使用Github Page、Bitbucket Page等空间。

  1. 创建远程仓库，username.github.io或username.bitbucket.org

  2. 初始化本地仓库，添加远程仓库，并拉取到本地

    git init
git remote add origin git@github.com:username/username.gitbucket.io.git master
git pull origin master:master

  3. 将Pelican生成出来的文件推送到远程仓库

    git push origin master:master

  4. 访问http://username.github.io/

  5. 自定义域名 向DNS提供商申请域名，创建CNAME记录定向到username.github.io 在远程仓库根目录创建CNAME文件
